2.1 词法分析

引言

词法分析，也称为扫描或词法分析器，是编译过程中的第一步。它负责将源代码（即程序员编写的程序）分解成一系列有意义的符号，这些符号被称为标记（tokens），可以叫做词元。 每个标记代表源代码中的一部分，如关键字、标识符、常量、运算符和分隔符。

在词法分析过程中，输入的源代码被逐字符地读取，并根据预定的规则（通常称为词法规则或正则表达式）进行匹配。当找到一个匹配时，就生成一个相应的标记，并将其发送到下一个编译阶段（通常是语法分析）。这个过程会一直持续到整个源代码文件都被处理完为止。

词法分析的主要目的是识别出源代码中的词汇结构，为后续的编译步骤提供基础。通过这种方式，编译器可以理解程序的基本元素，从而能够正确地解析和执行代码。

词法分析 的子部分

2.1.1 字符抽象

字符抽象是编译器在处理源代码时的一种重要机制。在编程中，我们使用的字符实际上可以被看作是特定的整数值（通常是ASCII值或者Unicode值）。编译器在读取源代码时，会将这些字符转换成对应的整数值，并进行相应的操作。

例如，在C语言中，我们可以使用字符字面量来表示单个字符，如’a’或'5’。编译器会将这些字符字面量解析为它们对应的整数值，并在需要的时候将其转换回字符形式。

此外，编译器还会对字符串进行特殊的处理。在C语言中，字符串实际上是字符数组，每个元素都包含了字符的整数值。当编译器遇到一个字符串字面量，如"Hello, world!"，它会创建一个包含所有字符的数组，并在末尾添加一个空字符（其整数值通常为0），以表示字符串的结束。

总的来说，字符抽象使得编译器能够有效地处理文本数据，并且不需要程序员关注底层的细节。通过这种方式，程序员可以更加专注于编写逻辑清晰、易于维护的代码。

好处

在词法分析中，对字符抽象的好处主要体现在以下几个方面：

  1. 简化处理逻辑：通过将输入的字符序列转换为一个更高级别的表示（如单词或符号），编译器可以忽略掉字符编码的细节，简化处理过程。例如，无论源代码使用的是ASCII还是Unicode编码，编译器都可以将其统一视为一个整数值流。
  2. 提高识别效率：通过对字符进行分类和归类，可以更快地识别出有意义的单词和符号。比如，在识别关键字时，只需要检查某个字符串是否与预定义的关键字列表匹配即可，而不需要关心每个字符的具体值。
  3. 便于错误检测：词法分析阶段通常会首先执行错误检测，如检查拼写错误、未闭合的括号等。使用字符抽象可以帮助编译器更容易地定位这些问题，并提供有用的错误信息。
  4. 减少后续阶段的工作量：对字符进行抽象后生成的词法单元是语法分析阶段的主要输入。如果在词法分析阶段就能够正确地区分不同类型的词素（如标识符、数字、运算符等），那么语法分析阶段就可以直接使用这些词素，而不需要重新处理原始的字符数据。
  5. 支持跨平台编译：对于不同的操作系统和硬件平台，可能使用的字符编码不同。通过在词法分析阶段进行字符抽象，编译器可以在一定程度上独立于具体的字符编码，从而提高代码的可移植性。
通过字符抽象，词法分析器能够以一种更加高效且平台无关的方式处理源代码，使得整个编译过程更加可靠和健壮。

2.1.2 字符流抽象

字符流抽象是指在编程和软件开发中，将底层的字节流或字符编码处理细节隐藏起来，提供一个更高级别的接口来操作文本数据。这样做的目的是简化程序设计，提高代码可读性，并使开发者可以专注于应用程序的逻辑和语义，而不是底层的实现细节。

在编译器中，词法分析阶段通常会对输入的源代码进行字符流抽象。这一阶段会将原始的字符序列转换为一系列有意义的单元，如标识符、关键字、数字、运算符等，这些单元被称为词法单元（或称为标记、符号）。每个词法单元都有对应的类型和值，这使得后续的语法分析和语义分析阶段能够更容易地理解和处理源代码。

此外，字符流抽象也经常出现在IO库的设计中，例如Java中的Reader和Writer类，或者C++中的iostream库。这些库提供了一个统一的方式来读取和写入字符流，无论底层的字符编码如何变化。使用这些库的开发者通常不需要关心字符编码的具体细节，从而可以更加高效地编写跨平台和兼容多种字符集的代码。

总的来说，字符流抽象是一种强大的工具，它可以帮助程序员更好地处理文本数据，而无需深入了解底层的字符编码细节。

好处

在词法分析中对字符流进行抽象有以下几个主要好处：

  1. 提高处理效率：对输入的字符流进行抽象，可以有效地将连续的字符组合成有意义的单元（如标识符、关键字、数字和运算符）。这有助于减少语法分析阶段需要处理的元素数量，从而提高编译器的整体效率。
  2. 更少的错误：通过将字符流转换为更高级别的表示，编译器能够更容易地检测出潜在的错误。例如，一个未闭合的字符串常量或注释可能会导致后续的解析错误；通过在词法分析阶段识别这些错误，可以提前报告并提供更准确的错误信息。
  3. 通用性与可移植性：字符流抽象使得编译器可以不关心具体的字符编码细节，比如ASCII、ISO-8859-1或者Unicode等。这种特性使得编译器能够在多种平台上运行，而无需针对每个平台进行特定的修改。
  4. 简化程序结构：词法分析器的设计和实现通常会利用字符流抽象的概念，比如使用有限自动机（FA）或正则表达式来匹配模式。这种方法简化了代码，使其更具可读性和维护性。
  5. 隐藏复杂性：对于程序员而言，他们通常关注于源代码的逻辑和语义，而非底层的字符编码。通过对字符流的抽象，编译器可以让程序员以一种更高层次的语言来表达意图，而不是直接操作底层的字符数据。
总的来说，字符流抽象是编译器设计中的重要概念，它能帮助编译器更好地理解源代码，并有效地生成目标代码。

 1// 字符串流
 2struct CharStream
 3{
 4    CharStream() {}
 5
 6    // 打开文件
 7    bool open(const std::string &path)
 8    {
 9        return readFile(path, data);
10    }
11
12    // 设置字符串数据
13    inline void setData(const std::string &str)
14    {
15        data = str;
16        index = 0;
17    }
18    inline std::string &getData() { return data; }
19    // 获取字符串中的一段
20    std::string getSbu(uint32_t start, uint32_t end)
21    {
22        assert(start < end);
23        return data.substr(start, end - start);
24    }
25
26    // 获取下一字符
27    Char next()
28    {
29        if (index < data.size())
30        {
31            uint32_t pos = index;
32            uint32_t ch = data[pos];
33            ++index;
34            return Char(pos, ch);
35        }
36        return Char(data.size(), 0);
37    }
38    // 退回字符
39    void back(Char ch)
40    {
41        index = ch.pos;
42        assert(index < data.size());
43    }
44    // 匹配一个字符
45    bool match(char ch, int off)
46    {
47        if (index + off > data.size())
48            return false;
49        return data[index + off] == ch;
50    }
51
52private:
53    std::string data; // 字符串数据
54    uint32_t index;   // 位置索引
55};

2.1.3 词法分析器

词法分析器（也称为扫描器、分词器或词法分析程序）是编译器或解释器的一部分，负责将输入的源代码字符串分解成一系列有意义的最小单元，这些单元被称为标记（Token）。每个标记代表了语言的一个原子单元，如关键字、标识符、常量、运算符和标点符号等。

在编译过程中，词法分析器通常是第一个执行的阶段。它从源代码的开头开始，按顺序读取字符，并根据语言的语法规则生成标记流。这些规则通常由正则表达式或者有限状态自动机来定义，它们描述了如何识别特定类型的标记。

词法分析器的主要任务包括：

  • 分隔：从连续的字符流中分离出独立的标记。
  • 标识化：确定每个标记的类型，并可能附带相关的属性值。
  • 错误处理：检测并报告源代码中的错误，如非法字符、未闭合的字符串或注释等。

例如，在C++编程语言中，以下源代码片段：

1int main() {
2    int x = 10;
3    return 0;
4}

词法分析器会将其分解为一系列标记：

标记 类型 值
int 关键字
main 标识符
( 符号
) 符号
{ 符号
int 关键字
x 标识符
= 符号
10 整数常量 10
; 符号
} 符号
return 关键字
0 整数常量 0
; 符号
} 符号

通过这种分割，后续的语法分析器能够更轻松地理解源代码的结构，并进行更复杂的语义检查和代码生成。

作用

词法分析器在编译过程中的主要作用如下：

  1. 标记化输入源代码：词法分析器将连续的字符序列分解成离散的标记（Token），这些标记代表了语言的基本元素，如关键字、标识符、常量和运算符等。
  2. 提供语法分析所需的输入：词法分析的结果通常是语法分析阶段的主要输入。语法分析器使用这些标记来构建抽象语法树（Abstract Syntax Tree, AST），进而理解程序的结构和语义。
  3. 错误检测：词法分析器可以检测源代码中的常见错误，如未闭合的字符串或注释、非法字符以及一些简单的拼写错误等。早期发现并报告错误有助于提高开发效率和程序质量。
  4. 简化后续处理：通过将复杂的字符流转换为易于处理的标记流，词法分析器降低了后续编译阶段的复杂性。这使得编译器设计者能够专注于更高层次的语法和语义问题。
  5. 增强可读性和可维护性：通过隐藏底层的字符编码细节，词法分析器提高了编译器代码的可读性和可维护性。开发者可以更轻松地理解和修改词法分析器，而不必关心低级别的实现细节。
  6. 支持跨平台编译：由于字符流抽象，词法分析器可以在一定程度上独立于具体的字符编码。这种特性使得编译器能够在多种操作系统和硬件平台上运行，而无需针对每个平台进行特定的修改。

总的来说，词法分析器是编译过程中的一个重要组成部分，它负责将复杂的字符数据转换为更容易理解和处理的形式，以便后续阶段对源代码进行深入分析和处理。

解析数字常量

  1Token Lexer::GetDigit()
  2{
  3    Token tok(TkKind::UIntLiteral);
  4    Char ch = stream.next();
  5    tok.pos.pos = ch.pos;
  6    tok.pos.line = pos.line;
  7    tok.addchar(ch.val);
  8
  9    if (ch.val == '0')
 10    {
 11        Char ch2 = stream.next();
 12        switch (ch2.val)
 13        {
 14        // 16进制
 15        case 'X':
 16        case 'x':
 17        {
 18            tok.addchar(ch2.val);
 19            while (true)
 20            {
 21                ch2 = stream.next();
 22                if (ch2.isHex() == false)
 23                {
 24                    stream.back(ch2);
 25                    break;
 26                }
 27                tok.addchar(ch2.val);
 28            }
 29
 30            if (tok.txt.empty())
 31                throw ErrorWhat::fmt("%u:%u: 不完整的 16 进制数：%s\n",
 32                                        tok.pos.line, tok.pos.pos, tok.txt.c_str());
 33            tok.val.i64 = std::strtoull(&tok.txt[2], nullptr, 16);
 34            tok.kind = TkKind::UIntLiteral;
 35            return tok;
 36        }
 37        break;
 38            // 2进制
 39        case 'B':
 40        case 'b':
 41        {
 42            tok.addchar(ch2.val);
 43            while (true)
 44            {
 45                ch2 = stream.next();
 46                if (ch2.isBin() == false)
 47                {
 48                    stream.back(ch2);
 49                    break;
 50                }
 51                tok.addchar(ch2.val);
 52            }
 53            if (tok.txt.empty())
 54                throw ErrorWhat::fmt("%u:%u: 不完整的 2 进制数：%s\n",
 55                                        tok.pos.line, tok.pos.pos, tok.txt.c_str());
 56            tok.val.i64 = std::strtoull(&tok.txt[2], nullptr, 2);
 57            tok.kind = TkKind::UIntLiteral;
 58            return tok;
 59        }
 60        break;
 61        // 浮点数
 62        case '.':
 63        {
 64            tok.addchar(ch2.val);
 65            return GetFltNumber(tok);
 66        }
 67        break;
 68        // 8 进制
 69        default:
 70        {
 71            while (true)
 72            {
 73                if (ch2.isOct() == false)
 74                {
 75                    stream.back(ch2);
 76                    break;
 77                }
 78                tok.addchar(ch2.val);
 79                ch2 = stream.next();
 80            }
 81            tok.val.i64 = std::strtoull(&tok.txt[1], nullptr, 8);
 82            tok.kind = TkKind::UIntLiteral;
 83            return tok;
 84        }
 85        break;
 86        }
 87    }
 88
 89    // 10进制或者浮点数
 90    while (true)
 91    {
 92        ch = stream.next();
 93        if (ch.isdigit() == false)
 94            break;
 95        tok.addchar(ch.val);
 96    }
 97    // 浮点数
 98    if (ch.val == '.')
 99    {
100        tok.addchar(ch.val);
101        return GetFltNumber(tok);
102    }
103    stream.back(ch);
104    // 10进制
105    tok.val.i64 = std::strtoull(tok.txt.c_str(), nullptr, 10);
106    return tok;
107}
108
109Token Lexer::GetFltNumber(Token &tok)
110{
111    tok.kind = TkKind::Flt64Literal;
112    Char ch = stream.next();
113    while (true)
114    {
115        if (ch.isdigit() == false)
116            break;
117        tok.addchar(ch.val);
118        ch = stream.next();
119    }
120    // 普通浮点数
121    if (ch.val != 'E' && ch.val != 'e')
122    {
123        // 32位浮点
124        if (ch.val == 'F' || ch.val == 'f')
125        {
126            tok.addchar(ch.val);
127            tok.kind = TkKind::Flt32Literal;
128            tok.val.f32 = std::strtof(tok.txt.c_str(), nullptr);
129            return tok;
130        }
131        // 64位浮点
132        stream.back(ch);
133        tok.val.f64 = std::strtod(tok.txt.c_str(), nullptr);
134        return tok;
135    }
136    // 科学计数法
137    tok.addchar(ch.val);
138    ch = stream.next();
139    // 查看符号位
140    if (ch.val == '+' || ch.val == '-')
141        tok.addchar(ch.val);
142    else if (ch.isdigit() == false)
143        throw ErrorWhat::fmt("%u:%u: 不完整的科学计数法：%s\n",
144                                tok.pos.line, tok.pos.pos, tok.txt.c_str());
145    while (true)
146    {
147        ch = stream.next();
148        if (ch.isdigit() == false)
149        {
150            stream.back(ch);
151            break;
152        }
153        tok.addchar(ch.val);
154    }
155    tok.val.f64 = std::strtod(tok.txt.c_str(), nullptr);
156    return tok;
157}

39                                        tok.pos.line, tok.pos.pos);

42                break;

解析操作符

  1Token Lexer::GetOperator()
  2{
  3    Token tok(TkKind::Operator);
  4    Char ch = stream.next();
  5    tok.pos.pos = ch.pos;
  6    tok.pos.line = pos.line;
  7    tok.addchar(ch.val);
  8    // 操作符
  9    switch (ch.val)
 10    {
 11        // 匹配 . 、 ...
 12    case '.':
 13    {
 14        if (stream.match('.', 0) == true && stream.match('.', 1) == true)
 15        {
 16            tok.kind = TkKind::Separator;
 17            ch = stream.next();
 18            tok.addchar(ch.val);
 19            ch = stream.next();
 20            tok.addchar(ch.val);
 21            tok.code.sp = TkSpEnum::Ellipsis;
 22            return tok;
 23        }
 24        tok.code.op = TkOpEnum::Dot;
 25        return tok;
 26    }
 27    break;
 28    // 匹配 + 、++、+=
 29    case '+':
 30    {
 31        if (stream.match('+', 0) == true)
 32        {
 33            ch = stream.next();
 34            tok.addchar(ch.val);
 35            tok.code.op = TkOpEnum::Plus2;
 36        }
 37        else if (stream.match('=', 0) == true)
 38        {
 39            ch = stream.next();
 40            tok.addchar(ch.val);
 41            tok.code.op = TkOpEnum::PlusAssign;
 42        }
 43        else
 44            tok.code.op = TkOpEnum::Plus;
 45        return tok;
 46    }
 47    break;
 48    // 匹配 - 、--、-=、->>、->>=
 49    case '-':
 50    {
 51        if (stream.match('-', 0) == true)
 52        {
 53            ch = stream.next();
 54            tok.addchar(ch.val);
 55            tok.code.op = TkOpEnum::Minus2;
 56        }
 57        else if (stream.match('=', 0) == true)
 58        {
 59            ch = stream.next();
 60            tok.addchar(ch.val);
 61            tok.code.op = TkOpEnum::MinusAssign;
 62        }
 63        else if (stream.match('>', 0) == true && stream.match('>', 1) == true)
 64        {
 65            ch = stream.next();
 66            tok.addchar(ch.val);
 67            ch = stream.next();
 68            tok.addchar(ch.val);
 69            if (stream.match('=', 0) == true)
 70            {
 71                ch = stream.next();
 72                tok.addchar(ch.val);
 73                tok.code.op = TkOpEnum::ShrAAssign;
 74            }
 75            else
 76                tok.code.op = TkOpEnum::ShrA;
 77        }
 78        else
 79            tok.code.op = TkOpEnum::Minus;
 80        return tok;
 81    }
 82    break;
 83    // 匹配 *、*=
 84    case '*':
 85    {
 86        if (stream.match('=', 0) == true)
 87        {
 88            ch = stream.next();
 89            tok.addchar(ch.val);
 90            tok.code.op = TkOpEnum::MultiplyAssign;
 91        }
 92        else
 93            tok.code.op = TkOpEnum::Multiply;
 94        return tok;
 95    }
 96    break;
 97    // 匹配 /、/=、单行注释、多行注释
 98    case '/':
 99    {
100        // 单行注释
101        if (stream.match('/', 0) == true)
102        {
103            ch = stream.next();
104            tok.addchar(ch.val);
105            return GetSinglineComment(tok);
106        } // 多行注释
107        else if (stream.match('*', 0) == true)
108        {
109            ch = stream.next();
110            tok.addchar(ch.val);
111            return GetMultilineComment(tok);
112        }
113        else if (stream.match('=', 0) == true)
114        {
115            ch = stream.next();
116            tok.addchar(ch.val);
117            tok.code.op = TkOpEnum::DivideAssign;
118        }
119        else
120            tok.code.op = TkOpEnum::Divide;
121        return tok;
122    }
123    break;
124    // 匹配 %、%=
125    case '%':
126    {
127        if (stream.match('=', 0) == true)
128        {
129            ch = stream.next();
130            tok.addchar(ch.val);
131            tok.code.op = TkOpEnum::ModulusAssign;
132        }
133        else
134            tok.code.op = TkOpEnum::Modulus;
135        return tok;
136    }
137    break;
138    // 匹配 &、&=、&&
139    case '&':
140    {
141        if (stream.match('=', 0) == true)
142        {
143            ch = stream.next();
144            tok.addchar(ch.val);
145            tok.code.op = TkOpEnum::BitAndAssign;
146        }
147        else if (stream.match('&', 0) == true)
148        {
149            ch = stream.next();
150            tok.addchar(ch.val);
151            tok.code.op = TkOpEnum::LogicalAnd;
152        }
153        else
154            tok.code.op = TkOpEnum::BitAnd;
155        return tok;
156    }
157    break;
158    // 匹配 |、|=、||
159    case '|':
160    {
161        if (stream.match('=', 0) == true)
162        {
163            ch = stream.next();
164            tok.addchar(ch.val);
165            tok.code.op = TkOpEnum::BitOrAssign;
166        }
167        else if (stream.match('|', 0) == true)
168        {
169            ch = stream.next();
170            tok.addchar(ch.val);
171            tok.code.op = TkOpEnum::LogicalOr;
172        }
173        else
174            tok.code.op = TkOpEnum::BitOr;
175        return tok;
176    }
177    break;
178    // 匹配 ~、~=
179    case '~':
180    {
181        if (stream.match('=', 0) == true)
182        {
183            ch = stream.next();
184            tok.addchar(ch.val);
185            tok.code.op = TkOpEnum::BitNotAssign;
186        }
187        else
188            tok.code.op = TkOpEnum::BitNot;
189        return tok;
190    }
191    break;
192    // 匹配 ^、^=
193    case '^':
194    {
195        if (stream.match('=', 0) == true)
196        {
197            ch = stream.next();
198            tok.addchar(ch.val);
199            tok.code.op = TkOpEnum::BitXOrAssign;
200        }
201        else
202            tok.code.op = TkOpEnum::BitXOr;
203        return tok;
204    }
205    break;
206
207    // 匹配 <、<=、<<、<<=、<<)、<<)=
208    case '<':
209    {
210        if (stream.match('=', 0) == true)
211        {
212            ch = stream.next();
213            tok.addchar(ch.val);
214            tok.code.op = TkOpEnum::LE;
215        }
216        else if (stream.match('<', 0) == true)
217        {
218            ch = stream.next();
219            tok.addchar(ch.val);
220            if (stream.match('=', 0) == true)
221            {
222                ch = stream.next();
223                tok.addchar(ch.val);
224                tok.code.op = TkOpEnum::ShlAssign;
225            }
226            else if (stream.match(')', 0) == true)
227            {
228                ch = stream.next();
229                tok.addchar(ch.val);
230                if (stream.match('=', 0) == true)
231                {
232                    ch = stream.next();
233                    tok.addchar(ch.val);
234                    tok.code.op = TkOpEnum::ShlRAssign;
235                }
236                else
237                    tok.code.op = TkOpEnum::ShlR;
238            }
239            else
240                tok.code.op = TkOpEnum::Shl;
241        }
242        else
243            tok.code.op = TkOpEnum::LT;
244        return tok;
245    }
246    break;
247
248    // 匹配 >、>=、>>、>>=、>>(、>>(=
249    case '>':
250    {
251        if (stream.match('=', 0) == true)
252        {
253            ch = stream.next();
254            tok.addchar(ch.val);
255            tok.code.op = TkOpEnum::GE;
256        }
257        else if (stream.match('>', 0) == true)
258        {
259            ch = stream.next();
260            tok.addchar(ch.val);
261            if (stream.match('=', 0) == true)
262            {
263                ch = stream.next();
264                tok.addchar(ch.val);
265                tok.code.op = TkOpEnum::ShrLAssign;
266            }
267            else if (stream.match('(', 0) == true)
268            {
269                ch = stream.next();
270                tok.addchar(ch.val);
271                if (stream.match('=', 0) == true)
272                {
273                    ch = stream.next();
274                    tok.addchar(ch.val);
275                    tok.code.op = TkOpEnum::ShrRAssign;
276                }
277                else
278                    tok.code.op = TkOpEnum::ShrR;
279            }
280            else
281                tok.code.op = TkOpEnum::ShrL;
282        }
283        else
284            tok.code.op = TkOpEnum::GT;
285        return tok;
286    }
287    break;
288
289    // 匹配 !、!=
290    case '!':
291    {
292        if (stream.match('=', 0) == true)
293        {
294            ch = stream.next();
295            tok.addchar(ch.val);
296            tok.code.op = TkOpEnum::NE;
297        }
298        else
299            tok.code.op = TkOpEnum::LogicalNot;
300        return tok;
301    }
302    break;
303    // 匹配 =、==
304    case '=':
305    {
306        if (stream.match('=', 0) == true)
307        {
308            ch = stream.next();
309            tok.addchar(ch.val);
310            tok.code.op = TkOpEnum::EQ;
311        }
312        else
313            tok.code.op = TkOpEnum::Assign;
314        return tok;
315    }
316    break;
317
318    } // 当作分隔符处理
319    return GetSeparator(tok, ch.val);
320}

解析分隔符

 1Token Lexer::GetSeparator(Token &tok, uint32_t val)
 2{
 3    tok.kind = TkKind::Separator;
 4    switch (val)
 5    {
 6        // 匹配 [
 7    case '[':
 8    {
 9        tok.code.sp = TkSpEnum::OpenBracket;
10        return tok;
11    }
12    break;
13    // 匹配 ]
14    case ']':
15    {
16        tok.code.sp = TkSpEnum::CloseBracket;
17        return tok;
18    }
19    break;
20    // 匹配 (
21    case '(':
22    {
23        tok.code.sp = TkSpEnum::OpenParen;
24        return tok;
25    }
26    break;
27    // 匹配 )
28    case ')':
29    {
30        tok.code.sp = TkSpEnum::CloseParen;
31        return tok;
32    }
33    break;
34    // 匹配 {
35    case '{':
36    {
37        tok.code.sp = TkSpEnum::OpenBrace;
38        return tok;
39    }
40    break;
41    // 匹配 }、};
42    case '}':
43    {
44        if (stream.match(';', 0) == true)
45        {
46            stream.next();
47            tok.addchar(';');
48            tok.code.sp = TkSpEnum::CBSemiColon;
49        }
50        else
51            tok.code.sp = TkSpEnum::CloseBrace;
52        return tok;
53    }
54    break;
55        // 匹配 :
56    case ':':
57    {
58        tok.code.sp = TkSpEnum::Colon;
59        return tok;
60    }
61    break;
62    // 匹配 ;
63    case ';':
64    {
65        tok.code.sp = TkSpEnum::SemiColon;
66        return tok;
67    }
68    break;
69    // 匹配 ,
70    case ',':
71    {
72        tok.code.sp = TkSpEnum::Comma;
73        return tok;
74    }
75    break;
76    default:
77        throw ErrorWhat::fmt("%u:%u: 未匹配词法规则的字符：%s\n",
78                                tok.pos.line, tok.pos.pos, tok.txt.c_str());
79        break;
80    }
81    return tok;
82}

解析单行注释

 1Token Lexer::GetSinglineComment(Token &tok)
 2{
 3    tok.kind = TkKind::SingleComments;
 4    Char ch = stream.next();
 5    while (true)
 6    {
 7        if (ch.isNull())
 8            break;
 9        else if (ch.isChar('\n'))
10        {
11            pos.line++;
12            break;
13        }
14        tok.addchar(ch.val);
15        ch = stream.next();
16    }
17    return tok;
18}

解析多行注释

 1Token Lexer::GetMultilineComment(Token &tok)
 2{
 3    tok.kind = TkKind::MultipleComments;
 4    Char ch = stream.next();
 5    while (true)
 6    {
 7        if (ch.isNull())
 8            throw ErrorWhat::fmt("%u:%u: 文件意外结束！缺少符号 ‘*/’\n",
 9                                    tok.pos.line, tok.pos.pos);
10        else if (ch.isChar('\n'))
11        {
12            pos.line++;
13            break;
14        }
15        tok.addchar(ch.val);
16        // 结束
17        if (ch.isChar('*') == true && stream.match('/', 0) == true)
18        {
19            ch = stream.next();
20            tok.addchar(ch.val);
21            break;
22        }
23        ch = stream.next();
24    }
25    return tok;
26}
